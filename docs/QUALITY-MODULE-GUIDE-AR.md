## 1. نظرة عامة ونموذج بيانات ISA-95

وحدة الجودة مبنية وفق معيار **ISA-95 الجزء الثالث**، الذي يحدد كيفية تدفق معلومات الجودة بين أنظمة المؤسسة (ERP) وأرضية التصنيع (MES).

### كائنات ISA-95 الأساسية المستخدمة

| مصطلح ISA-95 | الكائن في STAR-MES | الوصف |
|---|---|---|
| `QualityTestSpecification` | **خطة الجودة** | تحدد ماذا يُفحص وحدود القبول |
| `QualityTestSpecificationProperty` | **نقطة فحص / معامل قياس** | قياس مفرد مع حدود UCL/LCL/USL/LSL |
| `QualityTestResult` | **نتيجة الفحص** | القياس الفعلي أثناء الإنتاج أو بعده |
| `ProductionRequest` | **أمر العمل** | يربط نتيجة الفحص بتشغيل إنتاج محدد |
| `MaterialLot` | **دُفعة المادة** | دفعة قابلة للتتبع من مادة خام أو منتج نهائي |
| `NonConformanceReport` | **تقرير عدم المطابقة (NCR)** | يُوثق إخفاق الجودة وكيفية التصرف |
| `CorrectiveAction` | **إجراء تصحيحي (CAPA)** | الإجراءات التصحيحية والوقائية لمعالجة الجذر |

### أنواع الفحص الثلاثة وفق ISA-95

```
INCOMING   (وارد)       — فحص المواد عند استلامها في المخزن
IN_PROCESS (أثناء التصنيع) — فحص أثناء تنفيذ أوامر العمل النشطة
FINAL      (نهائي)      — فحص المنتج النهائي قبل الشحن
```

### شرح أنواع الحدود

```
UCL / LCL  — الحد الأعلى/الأدنى للسيطرة  → مُشتق من بيانات SPC (3 انحرافات معيارية)
USL / LSL  — الحد الأعلى/الأدنى للمواصفة → مُحدد من مواصفة المنتج أو متطلبات العميل

المنطقة بين UCL–USL وLCL–LSL = منطقة تحذير (مطابق للمواصفة لكن العملية تنجرف)
خارج USL/LSL = غير مطابق → يُنشئ تقرير عدم مطابقة (NCR)
خارج UCL/LCL = العملية خارج السيطرة الإحصائية → تنبيه SPC
```

## 3. خطوات إعداد خطط الجودة

**خطة الجودة** هي القالب الرئيسي الذي يحدد ما يُفحص. يجب إعداد خطة واحدة على الأقل قبل تشغيل أي فحص مرتبط بالإنتاج.

### 3.1 إنشاء خطة جودة

1. انتقل إلى **الجودة ← خطط الجودة**
2. انقر **خطة جديدة**
3. أكمل النموذج:

| الحقل | الوصف | مثال |
|---|---|---|
| رمز الخطة | معرف فريد (يُحوَّل تلقائياً لأحرف كبيرة) | `QP-RM-001` |
| نوع الفحص | متى تُستخدم هذه الخطة | `INCOMING` (وارد) |
| اسم الخطة | اسم يسهل فهمه | `فحص المواد الخام الواردة` |
| تكرار أخذ العينات | كم مرة يُجرى الفحص | `EVERY_BATCH` (كل دفعة) |
| كمية العينة | عدد الوحدات المأخوذة عينةً | `5` |
| الإصدار | رقم إصدار الخطة للتحكم في التغييرات | `1.0` |

4. انقر **إنشاء الخطة** — تُنشأ الخطة بحالة **مسودة**.

> **أنواع الفحص:**
> - `INCOMING` (وارد) — للمواد الخام عند الاستلام، فحص البضائع الواردة
> - `IN_PROCESS` (أثناء التصنيع) — فحوصات جانب الخط أثناء تنفيذ أوامر العمل
> - `FINAL` (نهائي) — فحص المنتج النهائي قبل الشحن

### 3.2 إضافة نقاط الفحص (المعاملات / Check Points)

بعد إنشاء الخطة، انقر عليها لفتح لوحة التفاصيل، ثم انقر **إضافة معامل**.

كل نقطة فحص تمثل خاصية قابلة للقياس:

| الحقل | إلزامي | الوصف | مثال |
|---|---|---|---|
| اسم المعامل | نعم | ما الذي يُقاس | `وزن التعبئة` |
| الوحدة | لا | وحدة القياس | `جرام` |
| القيمة الاسمية | لا | القيمة المستهدفة / المثالية | `500` |
| UCL | لا | الحد الأعلى للسيطرة (SPC) | `510` |
| LCL | لا | الحد الأدنى للسيطرة (SPC) | `490` |
| USL | لا | الحد الأعلى للمواصفة | `515` |
| LSL | لا | الحد الأدنى للمواصفة | `485` |
| طريقة الفحص | لا | كيفية إجراء القياس | `الوزن على ميزان معايَر` |
| معامل مؤشر الأداء (KPI) | لا | وضع علامة كمؤشر أداء رئيسي | `☑` |

**تسلسل الحدود:**

```
                   LSL                         USL
                    |                           |
         ←  مرفوض  |  LCL               UCL   |  مرفوض  →
                    |   |                 |    |
                    |   |   منطقة القبول  |    |
                    |   |                 |    |
                  485  490      500      510  515

                  LSL  LCL   القيمة الاسمية  UCL  USL
```

- القيم **خارج LSL/USL** ← رفض ← ينشئ تقرير NCR
- القيم **خارج LCL/UCL** لكن داخل LSL/USL ← مشروط (العملية تنجرف، تحقق)
- القيم **داخل UCL/LCL** ← قبول

### 3.3 اعتماد الخطة

يجب اعتماد خطط الجودة قبل اعتبارها فعّالة للإنتاج:

1. انقر زر **اعتماد** في بطاقة الخطة أو في لوحة التفاصيل
2. تتغير الحالة من **في انتظار الاعتماد** إلى **معتمدة**
3. الخطط المعتمدة فقط يمكن اختيارها عند تسجيل الفحوصات

> يجب استخدام الخطط المعتمدة فقط في فحوصات الإنتاج. الخطط في حالة المسودة للإعداد فقط.

### 3.4 اصطلاح تسمية الخطط (موصى به)

```
QP-{النوع}-{المنتج/الخط}-{الرقم التسلسلي}

أمثلة:
  QP-RM-SUGAR-001     → فحص وارد للسكر الخام
  QP-IP-LINE2-001     → فحص أثناء التصنيع للخط الثاني
  QP-FN-BETTI2L-001   → فحص نهائي لمنتج بيتي 2 لتر
```

## 4. خطوات تشغيل الفحوصات مع أوامر العمل

### 4.1 الربط وفق ISA-95

يمكن ربط كل فحص بـ:
- **أمر عمل** ← يربط نتيجة الجودة بتشغيل إنتاجي محدد
- **خطة جودة** ← تُحمِّل قائمة التحقق تلقائياً
- **آلة** ← يُسجل أي معدة أنتجت الدفعة
- **سجل دُفعة** ← التتبع الكامل من المادة الخام حتى المنتج النهائي

### 4.2 تسجيل فحص أثناء التصنيع (In-Process)

**السيناريو:** أمر العمل `WO-2026-0042` يعمل على الخط الثاني. يحتاج مهندس الجودة لتسجيل فحص منتصف الوردية.

1. انتقل إلى **الإنتاج ← أوامر العمل**
2. انقر على أمر العمل لفتح لوحة التفاصيل
3. انتقل لأسفل لقسم **فحوصات الجودة** — ستظهر جميع الفحوصات السابقة لهذا الأمر
4. انتقل إلى **الجودة ← الفحوصات**
5. انقر **فحص جديد**
6. أكمل النموذج:

| الحقل | القيمة | ملاحظة |
|---|---|---|
| النوع | `IN_PROCESS` | فحص أثناء الإنتاج |
| خطة الجودة | `QP-IP-LINE2-001` | اختر الخطة المناسبة |
| أمر العمل | `WO-2026-0042` | يربط الفحص بالأمر |
| الكمية الإجمالية | `50` | العينات المفحوصة |
| الكمية المقبولة | `48` | الوحدات التي اجتازت الفحص |
| الكمية المرفوضة | `2` | الوحدات التي فشلت |

7. بعد اختيار خطة الجودة، يظهر قسم **نقاط الفحص** تلقائياً بجميع المعاملات من الخطة
8. لكل معامل، أدخل:
   - **القيمة المقاسة** — القراءة الفعلية التي أُخذت
   - **قبول / رفض** — انقر الزر لتسجيل النتيجة
   - **ملاحظات** — أي مشاهدات

9. انقر **حفظ الفحص**

يحسب النظام تلقائياً:
- النتيجة الإجمالية: `PASS` (معدل نجاح ≥ 95%)، أو `CONDITIONAL` (80–95%)، أو `FAIL` (< 80%)
- يربط السجل بأمر العمل
- يحفظ جميع القياسات الفردية لتحليل SPC

### 4.3 عرض نتائج الجودة على أمر العمل

1. انتقل إلى **الإنتاج ← أوامر العمل**
2. انقر على أي أمر عمل
3. تُظهر لوحة التفاصيل اليمنى قسم **الجودة** يحتوي على:
   - جميع الفحوصات التي أُجريت لهذا الأمر
   - أعداد القبول/الرفض لكل فحص
   - شارة النتيجة (قبول / رفض / مشروط)
   - اسم خطة الجودة المرتبطة
   - اسم المفتش والتاريخ والوقت

### 4.4 تسجيل فحص وارد (استلام البضائع)

**السيناريو:** وصلت شحنة سكر خام. يحتاج أمين المخزن لفحصها وقبولها أو رفضها.

1. انتقل إلى **الجودة ← الفحوصات ← فحص جديد**
2. حدد النوع `INCOMING` (وارد)
3. اختر الخطة `QP-RM-SUGAR-001`
4. أدخل الكميات ونقاط الفحص
5. إذا كانت النتيجة **رفض (FAIL)**، يطالبك النظام برفع تقرير NCR

بدلاً من ذلك، ارتبط بـ **دفعة مادة** (من `المخزون ← دُفعات المواد`) للتتبع الكامل وفق ISA-95.

### 4.5 تسجيل فحص نهائي (قبل الشحن)

**السيناريو:** طبالي بيتي 2 لتر جاهزة للشحن. مطلوب توقيع قسم الجودة.

1. أنشئ فحصاً من النوع `FINAL` (نهائي)
2. اربطه بأمر العمل الذي أنتج هذه الطبالي
3. اختر خطة الفحص النهائي `QP-FN-BETTI2L-001`
4. أدخل القياسات لجميع نقاط الفحص
5. إذا كانت النتيجة `PASS` ← الدفعة مُخلَّصة للشحن
6. إذا كانت `FAIL` أو `CONDITIONAL` ← ارفع تقرير NCR قبل الإفراج

## 5. التكامل مع عملية التصنيع

وحدة الجودة مدمجة في عملية التصنيع على ثلاثة مستويات:

### 5.1 بوابات الجودة في دورة حياة أمر العمل

```
تسلسل حالات أمر العمل:
مخطط ← مُطلق ← قيد التنفيذ ← فحص الجودة ← مكتمل
                                      ↑
                        مطلوب نتيجة فحص هنا
```

خلال مرحلة `قيد التنفيذ`:
- يمكن للمشغلين/مهندسي الجودة تسجيل فحوصات **IN_PROCESS** في أي وقت
- تُظهر لوحة تفاصيل أمر العمل شارة حالة الجودة مباشرة

قبل التحويل إلى `مكتمل`:
- يجب تسجيل فحص **FINAL** (نهائي)
- في حال الرفض ← لا يمكن إكمال أمر العمل دون رفع تقرير NCR

### 5.2 ربط خطط الجودة بمنتجات (SKU)

يمكن ربط كل خطة جودة اختيارياً بـ **منتج (SKU)**، وهذا يعني:
- عند إنشاء أمر عمل لهذا المنتج، تُقترح خطة الفحص الصحيحة تلقائياً
- يمكن فلترة بيانات الفحص التاريخية حسب المنتج
- حسابات CPK في SPC تكون خاصة بالمنتج

للربط: حرر خطة الجودة ← أدخل معرف المنتج (SKU ID) في الإعدادات.

### 5.3 ربط خطط الجودة بالآلات

عند ربط خطة الجودة بـ **آلة محددة**:
- ترتبط فحوصات IN_PROCESS تلقائياً بتلك الآلة
- تظهر قياسات SPC على مخطط السيطرة الخاص بالآلة
- أحداث خروج العملية عن السيطرة يمكن أن تُطلق تنبيهات صيانة

### 5.4 تتبع الدفعات

كل فحص يمكن ربطه بـ **سجل دُفعة**، مما يُنشئ سلسلة تتبع كاملة وفق ISA-95:

```
دفعة مادة خام (وارد)
       ↓
أمر العمل (الإنتاج)
       ↓
سجل الدفعة (تفاصيل التشغيل)
       ↓
نتيجة الفحص (فحص الجودة)
       ↓
تقرير عدم المطابقة NCR (عند الرفض) → إجراء CAPA (تصحيحي)
       ↓
دفعة المنتج النهائي (صادر)
```

للوصول للتتبع الكامل: انتقل إلى **التتبع** في القائمة الجانبية.

## 6. سير عمل عدم المطابقة والإجراءات التصحيحية

### 6.1 متى تُرفع تقرير عدم مطابقة (NCR)؟

ارفع تقرير NCR عندما:
- تكون نتيجة الفحص **رفض (FAIL)**
- يكتشف المشغل عيباً أثناء الإنتاج
- يَرِد شكوى من عميل
- يكشف تدقيق داخلي عن انحراف في العملية

### 6.2 الحقول المطلوبة في تقرير NCR

| الحقل | الوصف |
|---|---|
| العنوان | وصف مختصر لحالة عدم المطابقة |
| الخطورة | `MINOR` (بسيط) / `MAJOR` (رئيسي) / `CRITICAL` (حرج) |
| فئة العيب | رمز الفئة (مثال: `LABELING`، `FILL_WEIGHT`، `SEAL`) |
| الكمية غير المطابقة | عدد الوحدات المتأثرة |
| الوصف | وصف تفصيلي — 10 أحرف على الأقل |
| تاريخ الاكتشاف | تاريخ ووقت اكتشاف العيب |
| تاريخ الاستحقاق | الموعد النهائي لحل المشكلة |
| الآلة | الآلة التي أنتجت العيب (اختياري) |

### 6.3 سير حالات تقرير NCR

```
مفتوح (OPEN)
     │
     ├─ قيد التحقيق
     │
     ▼
قيد المراجعة (IN_REVIEW)
     │
     ├─ تحليل السبب الجذري
     │
     ▼
في انتظار CAPA (CAPA_PENDING)
     │
     ├─ تأكيد السبب الجذري، رفع إجراء CAPA
     │
     ▼
محلول (RESOLVED)
     │
     ├─ تطبيق الحل والتحقق منه
     │
     ▼
مغلق (CLOSED)
```

**قواعد الانتقال بين الحالات:**
- `مفتوح ← قيد المراجعة` — تعيين لمهندس الجودة للتحقيق
- `قيد المراجعة ← في انتظار CAPA` — تحديد السبب الجذري، إنشاء إجراء CAPA
- `في انتظار CAPA ← محلول` — تطبيق الإجراء التصحيحي
- `محلول ← مغلق` — التحقق من الفعالية وانعدام التكرار

### 6.4 إنشاء CAPA من تقرير NCR

1. افتح تقرير NCR من **الجودة ← عدم المطابقة**
2. انقر **إضافة CAPA**
3. أكمل نموذج CAPA:

| الحقل | الوصف |
|---|---|
| النوع | `CORRECTIVE` (تصحيحي: معالجة السبب الجذري) أو `PREVENTIVE` (وقائي: منع تكرار مشكلات مشابهة) |
| العنوان | وصف مختصر للإجراء |
| الوصف | خطة العمل التفصيلية |
| الأولوية | `LOW` (منخفضة) / `MEDIUM` (متوسطة) / `HIGH` (عالية) / `CRITICAL` (حرجة) |
| المسؤول | الشخص المكلف بتنفيذ الإجراء |
| تاريخ الاستحقاق | الموعد النهائي للإنجاز |

4. أضف بنود الإجراء (مهام فرعية) للـ CAPA — لكل منها شخص مسؤول وموعد استحقاق ومتطلب إثبات
5. عند إتمام كل بند ← ضعه علامة "مكتمل" مع ملاحظات الإثبات
6. بعد اكتمال جميع البنود ← انقر **رفع للتحقق**
7. مدير الجودة يراجع وينقر **تحقق من الفعالية**
8. إذا كانت فعّالة ← **أغلق الـ CAPA**

## 7. ضبط العملية إحصائياً — SPC

### 7.1 ما الذي يفعله SPC؟

يُراقب SPC الاستقرار الإحصائي لعملية التصنيع. يرسم القياسات الفردية عبر الزمن وينبّه حين تخرج العملية عن السيطرة الإحصائية — **قبل** أن تبدأ المنتجات في الفشل.

### 7.2 كيف تُغذي القياسات نظام SPC؟

عند تسجيل فحص مع قيم مقاسة، تُخزَّن تلقائياً لتحليل SPC. يقوم النظام بـ:
- رسم كل قياس على مخطط السيطرة
- مقارنة القيمة مع UCL/LCL المحددة في معامل خطة الجودة
- تعليم النقاط الخارجة عن السيطرة
- حساب **Cpk** (مؤشر قدرة العملية)

### 7.3 قراءة قيم Cpk

```
Cpk ≥ 1.67  →  ممتاز — عملية قادرة بمستوى Six-Sigma
Cpk ≥ 1.33  →  جيد — يفي بمعظم معايير الجودة
Cpk ≥ 1.00  →  مقبول — العملية تحتاج تحسين
Cpk < 1.00  →  ضعيف — يُتوقع حدوث عدم مطابقة بشكل ملحوظ
```

### 7.4 الوصول إلى مخططات SPC

انتقل إلى **الجودة ← مخططات SPC**:
1. اختر معاملاً من القائمة المنسدلة
2. يُظهر المخطط: القياسات الفردية، خطوط UCL/LCL/المتوسط، والنقاط الخارجة عن السيطرة (محددة باللون الأحمر)
3. يُعرض Cpk بجانب المخطط

## 9. مرجع سريع لتعريفات الحقول

### حساب نتيجة الفحص

يحسب النظام نتيجة الفحص الإجمالية تلقائياً:

```
معدل النجاح = الكمية المقبولة ÷ الكمية الإجمالية × 100

PASS  (قبول)      → معدل النجاح ≥ 95%
CONDITIONAL (مشروط) → معدل النجاح 80% – 94%
FAIL  (رفض)      → معدل النجاح < 80%
```

### أنواع الفحص

| النوع | متى يُستخدم | المحفز النموذجي |
|---|---|---|
| `INCOMING` (وارد) | عند وصول المواد | استلام البضائع / تسليم أمر الشراء |
| `IN_PROCESS` (أثناء الإنتاج) | أثناء التصنيع | جدول المشغل / مهندس الجودة |
| `FINAL` (نهائي) | قبل الشحن | بوابة اكتمال أمر العمل |
| `PATROL` (جولة) | تدقيق عشوائي في الأرضية | جدول جولة الجودة |
| `AUDIT` (تدقيق) | تدقيق العملية / النظام | تدقيق داخلي أو خارجي |

### مستويات خطورة NCR

| الخطورة | التعريف | وقت الاستجابة |
|---|---|---|
| `MINOR` (بسيط) | انحراف مظهري أو بسيط، المنتج لا يزال قابلاً للاستخدام | 7 أيام |
| `MAJOR` (رئيسي) | انحراف وظيفي، المنتج قد لا يفي بالمتطلبات | 3 أيام |
| `CRITICAL` (حرج) | إخفاق في السلامة أو جودة حرجة، يُوقف الإنتاج | فوري |

### مستويات أولوية CAPA

| الأولوية | متى تُستخدم |
|---|---|
| `LOW` (منخفضة) | تحسين عملية بسيط، لا خطر فوري |
| `MEDIUM` (متوسطة) | خطر جودة لكن الإنتاج مستمر |
| `HIGH` (عالية) | عيب متكرر أو مشكلة تمس العميل |
| `CRITICAL` (حرجة) | مشكلة سلامة / تنظيمية، الإنتاج متوقف |

## 10. سيناريوهات شائعة وأمثلة عملية

### السيناريو أ: إطلاق منتج جديد — إعداد خطط الجودة

**الموقف:** يُطلق منتج جديد "بيتي 500 مل". يجب تعريف خطط الجودة قبل بدء الإنتاج.

1. **إنشاء خطة فحص وارد للمواد الخام:**
   - الرمز: `QP-RM-BETTI500-001`
   - النوع: `INCOMING`
   - نقاط الفحص: نقاء السكر (%)، نشاط الماء (Aw)، وزن التعبئة (جرام)
   - تعيين USL/LSL من مواصفة المنتج
   - اعتماد الخطة

2. **إنشاء خطة فحص أثناء الإنتاج:**
   - الرمز: `QP-IP-BETTI500-001`
   - النوع: `IN_PROCESS`
   - التكرار: `HOURLY` كل ساعة، كمية `5`
   - نقاط الفحص: وزن التعبئة (جرام)، قوة اللحام (N)، عزم الغطاء (N·cm)، موضع الملصق (mm)
   - تعيين UCL/LCL من بيانات SPC التاريخية لمنتجات مشابهة
   - اعتماد الخطة

3. **إنشاء خطة الفحص النهائي:**
   - الرمز: `QP-FN-BETTI500-001`
   - النوع: `FINAL`
   - نقاط الفحص: الوزن الصافي، درجة المظهر (1–5)، قراءة الباركود، سلامة الكرتون
   - اعتماد الخطة

---

### السيناريو ب: عيب متكرر في وزن التعبئة

**الموقف:** لاحظ مهندس الجودة انجراف قياسات وزن التعبئة نحو الانخفاض على مدى وردتين.

1. انتقل إلى **الجودة ← مخططات SPC**، اختر معامل "وزن التعبئة"
2. لاحظ أن آخر 6 قياسات تتجه نحو ما دون LCL
3. سجّل فحص **IN_PROCESS** ← النتيجة = `CONDITIONAL`
4. ارفع تقرير **NCR**: الخطورة `MAJOR`، الفئة `FILL_WEIGHT`، الكمية `120 وحدة`
5. حوّل NCR إلى `IN_REVIEW` ← حقق في معايرة آلة التعبئة
6. السبب الجذري: فوهة التعبئة مسدودة جزئياً
7. حوّل إلى `CAPA_PENDING`، أنشئ إجراء `CORRECTIVE`:
   - إجراء 1: تنظيف وإعادة معايرة فوهة #3 (الصيانة ← اليوم)
   - إجراء 2: إضافة فحص الفوهات لقائمة الصيانة الوقائية الأسبوعية (مدير الجودة ← هذا الأسبوع)
8. اكتمال الإجراءين ← رفع للتحقق
9. مراقبة مخطط SPC للـ 24 ساعة التالية ← لا تكرار ← إغلاق CAPA
10. تحويل NCR إلى `RESOLVED` ← `CLOSED`

---

### السيناريو ج: رفض مادة واردة

**الموقف:** وصلت دفعة من أغشية التغليف لكن سمكها خارج المواصفة.

1. استلام البضائع ← سجّل فحص **INCOMING** باستخدام الخطة `QP-RM-FILM-001`
2. قياس السمك: 3 عينات تقرأ 0.08 مم، 0.07 مم، 0.07 مم (LSL = 0.09 مم)
3. النتيجة = `FAIL` (رفض)
4. ارفع NCR: الخطورة `MAJOR`، الفئة `RAW_MATERIAL`، التصرف = `RETURN_TO_SUPPLIER` (إعادة للمورد)
5. وضع دفعة المادة في منطقة التخزين **Quarantine** (الحجر)
6. إبلاغ قسم المشتريات للتواصل مع المورد
7. يسير NCR عبر سير العمل ← يُغلق عند قبول المواد البديلة

```

  GET    /api/v1/quality/plans              سرد جميع الخطط مع المعاملات

  DELETE /api/v1/quality/plans/:id/parameters/:paramId     حذف معامل

  DELETE /api/v1/quality/inspections/:id      حذف فحص
  GET    /api/v1/quality/work-orders/:id/inspections   جميع فحوصات أمر عمل

  GET    /api/v1/quality/ncr                 سرد مع فلاتر

  GET    /api/v1/quality/capa                سرد مع فلاتر
```
