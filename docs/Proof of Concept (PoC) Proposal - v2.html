Executive Narrative

1. Executive Summary

In today’s high-volume manufacturing environment, unlocking capacity and improving Overall Equipment Effectiveness (OEE) requires real-time visibility. However, achieving this visibility in legacy environments often presents severe technical bottlenecks.

This Proof of Concept (POC) proposal outlines a rapid implementation strategy to deploy an Industry 4.0 Smart Factory solution on NCC’s Betti Line. Acknowledging the strict constraints of the existing Siemens and Omron PLCs, we propose a 100% Non-Intrusive IoT Overlay Architecture.

Instead of traditional, highly intrusive integration, our solution relies on an independent, air-gapped sensor network, edge computing, and cloud analytics. This approach entirely eliminates operational risk, requires zero machine downtime for integration, and delivers immediate actionable insights.

Strategically, this PoC is designed as Phase 1 of a broader application suite in which the connectivity and core MES layers are activated first, while simulation, AI intelligence, and enterprise integrations are unlocked in later deployment phases.

Constraint Landscape

2. Current State Constraints

An initial assessment of the Betti Line reveals constraints that render traditional MES integration highly risky:

  • Siemens S7-300 Environment

    Closed system architecture. Modifications are strictly prohibited, and communication availability is unguaranteed, blocking standard S7/OPC UA polling.

  • Omron Environment

    No free communication ports available on existing modules. Modifying the PLC program to add parallel communication logic is not permitted initially.

  • Business Constraint

    Zero tolerance for operational disruption or impact on current machine cycle times during the PoC phase.

Delivery Logic

5. Technical Approach & Sensors

Signal-Based Strategy

Because we cannot use a Tag-Based strategy initially, we employ a strictly Signal-Based approach.

  • Availability: Derived purely from Stack Light overlay.
  • Performance: Photoelectric sensor counts units against Ideal Cycle Time.

Accuracy Calibration

  • Debouncing: Edge layer software ignores pulses < 200ms apart to prevent double counts.
  • Micro-stops: Stoppages < 3 mins categorized as Performance loss. > 3 mins categorized as Availability loss.

Integration Requirements

12. Requirements for Direct PLC Integration

To unlock Layers 03 & 04 of the Application Suite (Simulation and AI), deeper machine integration is required. NCC will need to prepare the Omron PLCs for direct tag-based communication during a scheduled maintenance window.

Current State Limitations

  • • Existing Omron PLC communication ports are fully occupied.
  • • Overlay sensors cannot tell the system why the machine is down (specific fault codes).
  • • Process variables (temperature, exact motor speed) are locked inside the PLC.

Proposed Upgrade Approach

1

Hardware Expansion

Install an Omron Ethernet/IP or standard Ethernet expansion module.

2

Logic Modification

Map critical internal memory areas to network-accessible variables without altering core logic.

3

Direct Protocol Polling

Transition the Edge Gateway to natively polling the new Omron port.
